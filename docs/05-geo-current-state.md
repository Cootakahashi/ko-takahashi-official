# 05 - GEO（生成AI検索最適化）現状分析

## GEOとは

GEO (Generative Engine Optimization) は、AI検索エンジン（Perplexity、ChatGPT Search、Google AI Overview、Gemini、Copilot等）が質問に回答する際に、自サイトのコンテンツを引用・推薦するための最適化手法。

従来のSEOが「検索結果の上位表示」を目指すのに対し、GEOは「AIの回答に自分の情報が含まれること」を目指す。

## AI検索エンジンのクロール方式

| エンジン | JS実行 | 主な情報源 | クローラー |
|---------|--------|-----------|-----------|
| Google AI Overview | 部分的 | Google検索インデックス | Googlebot |
| Perplexity | なし | 独自クロール + Bing | PerplexityBot |
| ChatGPT Search | なし | Bing + 独自 | OAI-SearchBot |
| Gemini | なし | Google検索 | Google-Extended |
| Copilot | なし | Bing | Bingbot |

> **重要**: ほぼ全てのAI検索エンジンはJSを実行しない。SPAのコンテンツはAI検索から完全に不可視。

## 現状のGEO対応

### 実装済み

#### 1. llms.txt（基本レベル）
- ファイル: `/public/llms.txt`
- 内容: プロフィール概要、プロジェクト説明、スキル、リンク
- 課題:
  - LinkedInのURLが `YOUR_CUSTOM_ID_HERE`（未設定）
  - リンク先が `ko-takahashi-official.vercel.app`（間違い）
  - FAQ形式のQ&Aがない
  - 引用可能な短文ステートメントがない
  - 記事リストがない

#### 2. JSON-LD 構造化データ
- Person, Organization, FAQ スキーマは実装済み
- AIが構造化データを参照する場合に有効
- ただしSPAのため、AI検索ボットがJSON-LDを読み取れない可能性

#### 3. FAQ Schema（4問）
- 「高橋高とは誰ですか？」
- 「高橋高のプログラミングスキルは？」
- 「高橋高の会社は？」
- 「高橋高の経歴は？」

> Q&Aの数が少なく、カバー範囲が狭い

#### 4. noscript フォールバック
- 基本的なプロフィール情報のみ
- 物語やスキル詳細、記事コンテンツは含まれない

### 未実装

| 施策 | 説明 | 重要度 |
|------|------|--------|
| SSR/SSG | HTML直出しによるAIクローラー対応 | Critical |
| llms-full.txt | 詳細版AI向けドキュメント | High |
| /.well-known/ai-plugin.json | AIエージェント向け標準ファイル | Medium |
| 引用可能ステートメント | AIが引用しやすい短文の戦略的配置 | High |
| 事実・数字の構造化 | 定量データをAIが取得しやすい形に | High |
| 英語コンテンツ充実 | AI検索は英語コンテンツを圧倒的に多く引用 | High |
| 外部プラットフォームコンテンツ | 各プラットフォームからのシグナル強化 | Medium |
| @graph パターン | JSON-LDの統合・関係性明示 | Medium |

## GEOにおけるコンテンツ引用の仕組み

AI検索エンジンは以下のパターンを引用しやすい:

### 1. 定義文
```
「Culture OSとは、日本の伝統美学をデジタル基盤に統合するフレームワークである。」
```
→ **現状**: サイト内にこの形式の定義文が少ない

### 2. リスト形式の事実情報
```
高橋高の特徴:
- 11歳で学校を辞める
- 4ヶ国語（日本語、英語、中国語、タイ語）
- 23歳で破産、7年間の沈黙後に復活
- Jon & Coo Inc. CEO
```
→ **現状**: llms.txt に部分的にあるが不十分

### 3. 数字・統計
```
「7年間の独学期間」「4ヶ国語を習得」「2020年にJon & Coo設立」
```
→ **現状**: 散在しているが構造化されていない

### 4. FAQ形式
```
Q: 高橋高とは誰ですか？
A: 高橋高は新宿拠点の起業家で...
```
→ **現状**: 4問のみ。15-20問に拡充が必要

### 5. 第三者言及
他のサイト・プラットフォームから「高橋高」について言及されること。
→ **現状**: 外部プラットフォーム（Qiita, Zenn等）にコンテンツはあるが、サイトへの逆リンクが不明

## AIが「高橋高」について知っていること

現状、以下の理由でAI検索エンジンが「高橋高」について十分な情報を持っていない可能性が高い:

1. **サイトがSPA** → AIクローラーがコンテンツを読めない
2. **Wikipedia記事なし** → AIの最大の情報源がない
3. **外部メディア言及少** → E-E-A-Tシグナル不足
4. **英語コンテンツ不足** → グローバルAI検索でヒットしない
5. **ドメイン不整合** → 信号が分散

## GEO改善の方向性

```
Phase 1: 技術基盤（SSR化 + llms.txt強化）
  → AIクローラーがコンテンツを読める状態にする

Phase 2: コンテンツ構造化（FAQ拡充 + 定義文 + 数字）
  → AIが引用しやすい形にコンテンツを整備

Phase 3: 外部シグナル（プラットフォーム連携 + バックリンク）
  → 複数ソースからの一貫した情報発信

Phase 4: 英語コンテンツ（Medium + Dev.to + 英語ブログ）
  → グローバルAI検索での引用率向上
```
